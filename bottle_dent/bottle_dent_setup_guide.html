환경 준비 — conda 활성화

기존 lerobot 가상환경 재활용
① conda 환경 목록 확인
터미널 (NUC)
$ conda env list          # 설치된 conda 환경 목록
② lerobot 환경 활성화 (프롬프트가 (lerobot)으로 바뀜)
터미널 (NUC)
$ conda activate lerobot   # 가상환경 켜기
⚠️
conda 명령을 못 찾으면 (command not found) 아래로 PATH 등록:
터미널 (NUC) — 오류 시에만
$ ~/miniconda3/bin/conda init bash  # conda PATH 등록
$ source ~/.bashrc                 # 설정 다시 읽기
2

패키지 설치

3D 검사에 필요한 라이브러리
① 필요한 패키지를 한 번에 설치 (PyTorch 포함)
터미널 (NUC, lerobot 환경)
(lerobot)$ pip install pyrealsense2 torch flask opencv-python numpy --break-system-packages
# pyrealsense2 : RealSense 카메라 제어
# torch        : PyTorch (depth 텐서 연산, GPU 가속)
# flask        : 웹서버
# opencv-python: 영상 처리·컬러맵
# numpy        : 수치 배열 처리
💡
이미 설치된 패키지는 "already satisfied"로 건너뜁니다. --break-system-packages는 conda 환경 안에서 무해합니다.
② 설치 확인 (import 테스트)
터미널 (NUC)
(lerobot)$ python3 -c "import pyrealsense2, torch, flask, cv2; print('OK')"
✅ 정상 출력
OK
⚙️
GPU 자동 감지: 코드가 torch.cuda.is_available()로 GPU를 자동 확인합니다. NUC는 GPU가 없어 CPU로 동작하며, depth 비교는 가벼워서 CPU로 충분합니다.
3

파일 배치

소스를 NUC 작업 폴더에 배치
① 작업 폴더 생성 및 이동
터미널 (NUC)
(lerobot)$ mkdir -p ~/bottle_dent      # 작업 폴더 생성
(lerobot)$ cd ~/bottle_dent           # 폴더로 이동
# 다운로드한 파일들을 이 폴더에 복사 (USB/scp 등)
② 폴더 구조 확인 (templates 안에 index.html, golden 폴더 필수)
bottle_dent/
├── app.py                # Flask 웹서버 (메인 실행)
├── realsense_camera.py   # RealSense RGB+Depth 캡처
├── dent_detector.py      # 찌그러짐 검출 (PyTorch depth 비교)
├── test_camera.py        # 카메라 연결 테스트
├── requirements.txt      # 패키지 목록
├── templates/
│   └── index.html        # 웹 화면
└── golden/               # 양품 기준 depth 저장 폴더
③ 파일 확인
터미널 (NUC)
(lerobot)$ ls -R ~/bottle_dent       # 폴더 내용 확인
⚙️
설정값 확인: app.py 상단의 설정값을 환경에 맞게 조정하세요. (다음 표 참고)
설정값기본값설명·조정
DENT_THR_MM5.0찌그러짐 판정 깊이(mm). 30cm 거리 기준. 멀면 ↑
MIN_AREA_PX80찌그러짐 최소 면적. 작은 노이즈 무시 기준
GOLDEN_FRAMES5양품 등록 시 평균낼 장수
FLASK_PORT5000웹서버 포트

카메라 점검

실행 전 D435i 정상 확인
① OS가 카메라를 인식하는지 확인

(lerobot)$ rs-enumerate-devices | head  # 연결된 RealSense 목록
🔌
장치가 안 보이면 USB3 포트(파란색)에 직접 연결됐는지 확인. USB2나 허브는 대역폭 부족으로 불안정합니다.
② 테스트 스크립트로 프레임 수신 확인
터미널 (NUC)
(lerobot)$ python3 test_camera.py      # 카메라 연결·프레임 테스트
✅ 정상 출력 예시
🔍 RealSense 연결 테스트 시작...
✅ 카메라 연결 성공 | depth_scale = 0.001
  프레임1: 컬러 (480, 848, 3), 중앙거리 0.312m
  ...
✅ 테스트 완료 — 카메라 정상! app.py를 실행하세요.

Usage · 검사 사용법

🖥️ 웹 화면에서 검사하는 순서

화면은 ① 양품 등록 → ② 검사 대상 촬영 → ③ 비교 흐름입니다. 처음엔 두 이미지가 비어있고, 단계대로 채워집니다.

1
양품 5장 등록 (STEP 1) 정상 생수병을 카메라 앞 30cm에 놓고 한장 3D 촬영을 5번 누르거나(진행바 5/5), 연속 5장 3D 촬영 1번. 5장 평균이 왼쪽에 채워집니다.
2
검사 대상 촬영 (STEP 2) 검사할 병으로 바꿔 같은 위치에 놓고 현재 3D 촬영. 오른쪽에 검사 대상이 채워집니다.
3
비교 실행 (STEP 3) 양품·현재가 둘 다 채워지면 비교 실행 버튼이 활성화됩니다. 누르면 판정(PASS/BLOCK)·찌그러짐 깊이(mm)·위치·히트맵이 나옵니다.
📌
핵심: 양품 등록이 먼저입니다. 3D 검사는 "양품 대비 차이"로 판정하므로, 양품을 등록하지 않으면 비교가 안 됩니다. 양품·검사품의 위치·각도를 같게 해야 정확합니다.
Accuracy · 정확도 팁

🎯 정확한 검사를 위한 핵심 수칙

항목권장
카메라-병 거리30~40cm (최소 28cm 이상)
검출 가능 찌그러짐5mm 이상 (30cm 기준 오차 고려)
카메라 워밍업전원 후 5~10분
카메라 고정삼각대로 완전 고정, 등록 후 이동 금지
병 종류불투명 또는 라벨 부위 (투명 페트병 주의)
조명균일 실내광, 직사광선 금지
💧
투명병 주의: 적외선 depth 센서는 투명 페트병을 통과·산란시켜 측정이 군데군데 빕니다. 코드에 "유효 비율" 경고를 넣어 50% 미만이면 자동 알림이 뜹니다. 교육 첫 실습은 불투명 병이나 라벨 부위를 권장합니다.
Troubleshooting · 문제 해결

🔧 자주 발생하는 문제

Q. conda: command not found

PATH 미등록. ~/miniconda3/bin/conda init bash 후 source ~/.bashrc.

Q. 카메라가 rs-enumerate-devices에 안 보임

USB3 포트(파란색) 직결 확인. 허브·연장선 제거.

Q. RuntimeError: Frame didn't arrive / Device busy

realsense-viewer 등 다른 프로그램이 카메라 점유 중. 종료 후 재실행.

Q. 라이브 화면이 "Waiting for camera..."에서 안 바뀜

프레임 미수신. test_camera.py로 먼저 점검, USB3 연결 확인.

Q. 비교 버튼이 회색(비활성)

양품 등록 + 현재 촬영이 둘 다 완료돼야 활성화됩니다. 상단 단계 표시(STEP) 확인.

Q. 정상 병인데 자꾸 BLOCK

① 양품·검사품 위치가 다름 → 같은 자리에 ② 임계값이 낮음 → DENT_THR_MM을 6~7로 ③ 카메라가 움직임 → 재고정 후 양품 재등록.

Q. 측정 품질 낮음 경고가 뜸

투명병이거나 거리·조명 문제. 불투명 병/라벨 부위로 바꾸고 거리 30~40cm 확인.

Q. 브라우저에서 NUC IP 접속 안 됨

방화벽이 포트 막을 수 있음. sudo ufw allow 5000 또는 NUC 본체에서 localhost:5000 접속.
