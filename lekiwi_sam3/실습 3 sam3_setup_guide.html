박스(NUC)는 "이 네모 안에 물건이 있어요"라고 알려주는 대충 표시예요. 마스크(4090)는 가위로 물건 모양 그대로 정확히 오려내는 거예요. 비스듬히 놓인 기판을 네모로 감싸면 모서리에 배경이 잔뜩 들어가지만, 가위로 오리면 기판만 깔끔하게 남아요.

단계별 따라 하기

각 카드의 명령어를 복사해서 그대로 실행하세요. 어디서(4090/NUC) 실행하는지 카드마다 적어두었어요.

1 🔌
NUC → 4090 연결 확인NUC에서 실행 · 두 컴퓨터가 서로 통하는지
$curl -m 3 http://192.168.0.75:5001/ping
▶ 이렇게 하면

NUC가 4090에게 "거기 있니?" 하고 신호를 보내요.

1{"status":"ok"} 가 나오면 두 컴퓨터가 잘 연결된 거예요.
2아무 응답이 없거나 멈추면 IP·포트·방화벽을 확인해요. (192.168.0.75는 4090의 실제 IP로 바꾸세요)
💡팁: 4090 IP는 4090에서 hostname -I 로 확인해요. NUC와 같은 192.168.0.x 대역이어야 통해요.
2 🧪
SAM3 단독 테스트4090에서 실행 · 서버 켜기 전에 모델만 먼저 확인
$conda activate sam3 && python test_sam3.py
▶ 이렇게 하면

SAM3 모델이 혼자서도 잘 도는지 먼저 확인해요. (Flask 서버 없이)

1로컬에 있는 sam3.pt 파일을 불러와요. (인터넷 다운로드 안 함)
2가짜 이미지에 "object"라고 물어보고 마스크가 나오는지 봐요.
3아래처럼 "마스크 생성 성공"이 뜨면 3번으로 넘어가요.
[1] sam3 import 성공 [2] 로컬 모델 로드 중 (load_from_HF=False)... [3] 모델 로드 성공 (다운로드 없이!) [4] set_image 성공 [5] set_text_prompt 성공 >>> 마스크 생성 성공! 개수=1, shape=(1, 480, 640)
💡왜 먼저 테스트하나요? 모델이 문제인지, 서버 코드가 문제인지 나눠서 확인하려고요. 여기서 성공하면 모델은 정상이에요.
3 🎯
4090 SAM3 서버 켜기4090에서 실행 · 정밀분할을 담당하는 서버
$conda activate sam3 && python sam3_server.py
▶ 이렇게 하면

4090이 "마스크 만들어 줄게" 하고 5001번 문을 열고 기다려요.

1SAM3 모델을 GPU에 올려요. (2번 테스트가 됐으면 여기도 잘 돼요)
2http://0.0.0.0:5001 에서 NUC의 요청을 기다려요.
⚠️주의: 이 창은 끄지 말고 계속 켜두세요. 끄면 NUC가 "SAM3 오프라인"이라고 표시해요.
4 🖥️
NUC 메인 서버 켜기NUC에서 실행 · 카메라·YOLO·GDINO·화면 담당
$conda activate lerobot && python app.py
▶ 이렇게 하면

NUC가 화면을 띄우고 4090과 연결돼요.

1YOLO·GDINO 모델을 불러오고 카메라 영상을 받아요.
2http://0.0.0.0:5000 웹 화면이 열려요.
3시작 메시지에 4090 주소(SAM3_SERVER_URL)가 맞는지 확인해요.
💡팁: app.py 위쪽 SAM3_SERVER_URL 값이 4090 실제 IP(http://192.168.0.75:5001)인지 먼저 확인하세요.
5 🔍
브라우저에서 정밀분할아무 PC의 브라우저 · 박스 vs 마스크 비교
›http://<NUC_IP>:5000
▶ 이렇게 하면

물체를 올리고, 영어로 찾고, 정밀분할 버튼을 누르면 비교 결과가 나와요.

1카메라 앞에 기판 같은 물체를 비스듬히 올려요.
2입력창에 circuit board 같은 영어 단어를 넣고 "박스로 위치 찾기"를 눌러요.
3"정밀분할 실행"을 누르면 왼쪽 빨강 박스 / 오른쪽 초록 마스크가 나란히 나와요.
4"박스의 N%가 배경이었습니다" 숫자로 정밀분할 효과를 확인해요.
📊
무엇을 배우나요

비스듬한 기판일수록 박스에 배경이 많이 들어가서 "배경 %"가 커져요. 마스크는 그걸 깔끔하게 걷어내죠. 로봇이 물체를 집을 때 이 차이가 정확도를 좌우해요.

🩺 막혔을 때 — 우리가 실제로 겪은 문제들

이 시스템을 만들며 실제로 만난 오류와 해결책이에요. 같은 증상이 보이면 여기를 보세요.

증상원인해결
헤더에 "SAM3 오프라인" NUC가 4090을 못 찾음 SAM3_SERVER_URL을 4090 실제 IP로. 같은 192.168.0.x 대역인지, 방화벽 5001 열렸는지 확인
연결은 됐는데 버튼 누르면 "SAM3 서버 응답 없음" /ping은 되지만 /segment 추론에서 에러 4090 터미널 로그 확인. 십중팔구 아래 두 줄 중 하나
mat1 and mat2 must have the same dtype, BFloat16 and Float 모델은 bfloat16인데 입력이 float32 torch.autocast("cuda", dtype=torch.bfloat16) 안에서 추론 (새 코드에 반영됨)
sam3.pt 0% 에서 멈춤 (다운로드 안 됨) load_from_HF=True라 로컬 파일 두고 HF에서 3.45GB 받으려다 멈춤 checkpoint_path="sam3.pt" + load_from_HF=False (새 코드에 반영됨)
GDINO 탐지가 9초나 걸림 NUC는 CPU뿐 → Grounding DINO가 느림 정상 범위. 첫 호출은 워밍업으로 더 느림. 타임아웃 30초로 늘려둠
bpe_simple_vocab_16e6.txt.gz 없음 일부 sam3 설치에서 BPE 토크나이저 파일 누락 4090 폴더에 그 파일을 받아두면 코드가 자동으로 찾음 (아래 명령 참고)
+ 📥
BPE 파일이 없다는 에러가 날 때4090에서 실행 · 작은 파일 하나만 받기
$curl -sL -o bpe_simple_vocab_16e6.txt.gz https://github.com/openai/CLIP/raw/main/clip/bpe_simple_vocab_16e6.txt.gz
▶ 이렇게 하면

SAM3가 글자를 이해하는 데 필요한 작은 사전 파일을 받아요.

1sam3_server.py가 있는 폴더에서 실행해요.
2받은 뒤 다시 test_sam3.py를 실행하면 자동으로 이 파일을 찾아 써요.
