2b✅
카메라 동작 테스트확인한 장치 번호로 실제 프레임이 읽히는지 테스트
# 모든 video 장치를 하나씩 열어서 테스트
Pi$ python3 -c "
import cv2
for i in range(5):
    cap = cv2.VideoCapture(f'/dev/video{i}', cv2.CAP_V4L2)
    ret, frame = cap.read()
    if ret:
        print(f'/dev/video{i}: ✅ OK  크기={frame.shape}')
    else:
        print(f'/dev/video{i}: ❌ 안됨')
    cap.release()
"
▶ 이렇게 하면

/dev/video0부터 video4까지 전부 테스트해서 어느 번호가 실제로 작동하는지 알 수 있어요!

# 출력 예시
/dev/video0: ❌ 안됨               ← 메타데이터 전용 (무시)
/dev/video1: ✅ OK  크기=(480, 640, 3)  ← FRONT로 사용!
/dev/video2: ✅ OK  크기=(480, 640, 3)  ← TOP으로 사용!
/dev/video3: ❌ 안됨               ← 메타데이터 전용 (무시)
/dev/video4: ❌ 안됨
📝스크립트 수정 방법: 위에서 ✅ OK인 번호를 스크립트에 넣으세요!

왜 2개로 분리하나요?

한 사무실(프로세스)에서 2명이 동시에 전화(카메라)하면 서로 방해돼요. 각자 다른 사무실에서 전화하면 문제없어요! Python의 GIL 잠금 + USB 경합을 피하는 방법이에요.

⚠️장치 경로 확인! --device 뒤에 7번에서 확인한 실제 경로를 넣으세요. 잘못된 경로면 카메라 열기 실패!

▶ 2대 동시 사용 시 FPS가 낮은 이유

Pi5의 USB 대역폭 경합 때문이에요. 두 카메라가 같은 USB 버스를 나눠 써요.

증상원인해결법
FRONT 10fps, TOP 5fpsUSB 대역폭 경합다른 USB 포트 종류에 연결
한쪽만 0fps장치 경로가 틀림v4l2-ctl --list-devices 재확인
양쪽 다 느림같은 USB 2.0 버스해상도 320×240으로 낮추기
간헐적 프레임 드롭Wi-Fi 지터NUC 유선 + Pi 5GHz 전용
💡현실적 목표: 2대 동시 사용 시 각 카메라 10fps면 교육 현장에서 충분해요. FRONT(YOLO용)를 우선하고 TOP은 5~7fps도 OK!
🔌USB 포트 분리 배치:
• 카메라 A → Pi5 USB 3.0 포트 (파란색)
• 카메라 B → Pi5 USB 2.0 포트 (검은색)
→ 서로 다른 USB 버스를 사용하여 경합 감소!
