Pi 듀얼 카메라 → NUC YOLO26 + BoT-SORT 실시간 트래킹 + Flask 웹 서버

🧠 왜 Pi가 아니라 NUC에서 AI를 돌리나?

Pi는 카메라만 담당하고, 무거운 AI는 NUC가 맡습니다.

📮
쉽게 말하면

Pi는 우체부예요. 편지(카메라 영상)를 모아서 보내기만 합니다. NUC는 본사예요. 받은 편지를 분석하고 판단합니다. 우체부에게 분석까지 시키면 배달이 느려지듯, Pi에서 AI를 돌리면 2fps밖에 안 나와요!

항목Pi 5 (ARM CPU)NUC i7 (Intel CPU)
OpenVINO YOLO26n❌ 사용 불가 (Intel 전용)✅ ~35ms (28fps)
PyTorch YOLO26n~500ms (2fps)~200ms (5fps)
BoT-SORT CMC❌ 너무 느림✅ orb 방식으로 빠름
역할📷 카메라 캡처 + 전송🧠 AI 추론 + 트래킹

📡 PHASE A — Pi 카메라 준비

Pi에서 카메라를 준비합니다

Pi에 SSH 접속 후 터미널 2개에서 실행하세요.

1🔍
카메라 장치 확인Pi에서 연결된 카메라 목록 보기
$v4l2-ctl --list-devices
▶ 이렇게 하면

Pi에 연결된 카메라 목록과 장치 번호가 나와요.

1USB 웹캠 2개가 보이는지 확인. FRONT는 보통 /dev/video0, TOP은 /dev/video2 또는 /dev/video3.
⚠️"카메라 열기 실패"가 나오면 이전 프로그램이 점유 중!
sudo kill $(sudo fuser /dev/video0 2>/dev/null | awk '{print $1}')
2📷
FRONT 카메라 전송Pi 터미널 1에서 실행 — 이 터미널은 열어두세요!
$conda activate lerobot && python send_camera_front.py
▶ 이렇게 하면

FRONT 카메라가 포트 5556으로 영상을 보내기 시작해요 (15fps).

3📷
TOP 카메라 전송Pi 터미널 2에서 실행 (새 터미널!)
$conda activate lerobot && python send_camera_top.py
▶ 이렇게 하면

TOP 카메라가 포트 5557로 영상을 보내기 시작해요.

1장치 번호가 다르면 옵션 추가: --device /dev/video2
📺
쉽게 말하면

TV 방송국에서 카메라 2대를 동시에 켜는 것과 같아요. FRONT는 채널 5556, TOP은 채널 5557로 송출!

🖥️ PHASE B — NUC 환경설정 (처음 한 번만)

NUC에서 환경을 준비합니다

이미 완료했으면 건너뛰세요.

4🌐
Pi IP 주소 확인 & 설정매번 바뀔 수 있어요!
[Pi에서]$hostname -I
▶ 이렇게 하면

Pi의 실제 IP가 표시돼요 (예: 192.168.50.111).

1NUC에서 코드 안의 IP를 수정 (두 파일 동시에):
[NUC]$sed -i 's/기존IP/새IP/g' receive_and_track.py app.py
⚠️DHCP 주의: 공유기가 IP를 자동 할당하므로 Pi 재부팅 시 IP가 바뀔 수 있어요. 매번 확인!
5🔎
YOLO 모델 경로 확인OpenVINO 변환 모델 위치 찾기
$find ~ -name "yolo26n_openvino_model" -type d 2>/dev/null
▶ 이렇게 하면

YOLO26n OpenVINO 모델의 실제 경로가 나와요.

1결과 경로를 코드의 YOLO_MODEL 변수에 반영하세요.
2모델 없으면: python -c "from ultralytics import YOLO; YOLO('yolo26n.pt').export(format='openvino', imgsz=640)"
🚨실제 사례: 코드에 /home/zetabank/인데 실제가 /home/zeta/면 FileNotFoundError!
6📦
패키지 설치 & BoT-SORT 설정필요한 패키지와 트래커 설정 파일
$conda activate lerobot && pip install flask pyzmq --break-system-packages
▶ 이렇게 하면

Flask(웹 서버)와 ZeroMQ(영상 수신)가 설치돼요.

1BoT-SORT 설정 파일도 만들어요 — model: yolo26n.pt 줄이 반드시 있어야 해요!
🚨실제 사례: model: 줄이 빠지면 AttributeError: has no attribute 'model' 에러! ultralytics 최신 버전에서 필수 필드.
7📂
templates 폴더 & 웹 UI 배치Flask가 HTML을 찾는 경로에 배치
$mkdir -p ~/ai/templates
▶ 이렇게 하면

templates/ 폴더가 생겨요. ZIP의 templates/index.html을 여기에 넣으세요.

🚨실제 사례: 이 폴더가 없으면 브라우저에서 500 Internal Server Error! Flask의 render_template()은 반드시 templates/ 폴더 안에서 HTML을 찾아요.

▶ 이렇게 하면

NUC 화면에 YOLO + BoT-SORT 트래킹 영상이 나와요!

키기능설명
t트래커 전환BoT-SORT ↔ ByteTrack
c카메라 전환FRONT → TOP → DUAL
p사람 필터사람만 추적 On/Off
s스크린샷현재 화면 JPG 저장
q종료프로그램 종료
⚠️중요: 키보드는 영상 창을 마우스로 클릭한 뒤 눌러야 해요!
9🔄
BoT-SORT vs ByteTrack 비교t키로 전환 후 카메라를 흔들어보세요
[영상 창 클릭]t 키 → 카메라 좌우로 흔들기
▶ 이렇게 하면

트래커가 전환돼요. 카메라를 흔들면서 차이를 비교하세요!

1BoT-SORT: 카메라 흔들어도 ID 유지 (CMC가 보정).
2ByteTrack: 카메라 흔들면 ID가 바뀌거나 끊김!
🚌
쉽게 말하면

버스에서 창밖 사람을 지켜볼 때, ByteTrack은 버스가 움직이면 "사람이 사라졌다!"고 혼란. BoT-SORT는 "버스가 움직인 거구나"라고 보정해서 계속 추적. 이게 CMC!

⚡ 왜 이렇게 빠른가? — 속도의 비밀 3가지

GPU 없이도 실시간 트래킹이 되는 이유입니다.

🚀
OpenVINO 가속
PyTorch: ~200ms (5fps)
~35ms (28fps)
Intel i7의 AVX-512 명령어를 활용. PyTorch보다 6배 빠른 추론!
📡
ZeroMQ CONFLATE
버퍼 쌓임: 딜레이 증가
딜레이 0ms
항상 최신 프레임만 처리. 버퍼에 쌓이지 않아 지연 없음!
📶
전용 네트워크
공용 Wi-Fi: 20~180ms
수신 ~0ms
전용 공유기(AX1800) + NUC 유선 + Pi 5GHz Wi-Fi!
🏎️
쉽게 말하면

OpenVINO는 고속도로(Intel 전용 최적화), CONFLATE는 우체통에 최신 편지만 남기기, 전용 공유기는 다른 차 없는 전용 차선이에요!
