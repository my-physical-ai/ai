VLM은 "판단하는 모델"이지 "찾는 모델"이 아닙니다. 좌표를 요구하면 안 되고, "이 이미지에 결함이 있나?"라고 물어야 합니다. 찾는 건 YOLOE(빠르게)와 SAM3(정밀하게)가 합니다.

⚡ 3. YOLO26 → YOLOE-26 전환 — 왜 바꿨나?

YOLO26 (80종 고정)

"사진 암기" 방식 — COCO 데이터셋 80종의 특정 모양만 인식

스타벅스 컵 → "refrigerator" 오인 (학습한 컵 사진과 안 닮아서)

YOLOE-26 (Open Vocabulary)

"개념 이해" 방식 — 영어 단어의 의미를 이해하여 매칭

스타벅스 컵 → "cup" 정확 탐지 ("마시는 용기" 개념으로 이해)

📚
비유: 시험 공부법

YOLO26 = 기출문제 80개만 외운 학생 → 새 문제 나오면 틀림.
YOLOE-26 = 개념을 이해한 학생 → 37개만 배워도 응용 가능.

🎯 4. SAM3 — 텍스트 vs 박스 프롬프트 시행착오

문제 1: "tumbler"를 모른다
SAM3의 set_text_prompt("tumbler") → "찾지 못했습니다" 에러. SAM3가 모든 영어 단어를 아는 게 아님.
해결 시도: SAM3_LABEL_MAP 매핑
"tumbler"→"bottle", "package"→"box" 변환 테이블. 동작했지만 근본 해결은 아님.
시도: add_geometric_prompt (박스 프롬프트)
"이름이 왜 필요해? 위치만 주면 되잖아!" → 맞지만 마스크가 박스 전체를 채움 (100% 정확도 = 분할 안 됨).
최종 결정: 텍스트 프롬프트 + LABEL_MAP 유지
set_text_prompt("bottle")가 가장 정밀한 윤곽을 그림. YOLOE 라벨을 매핑해서 전달.
🔬
SAM3 cutout의 가치

SAM3 마스크로 배경을 흰색으로 제거한 cutout 이미지를 VLM에 전달하면, VLM이 배경에 신경 쓰지 않고 물체 표면만 집중 분석합니다. 이 cutout을 확대하면 미세한 결함도 감지 가능.

🧠 5. VLM 프롬프트 엔지니어링 — 핵심 교훈 5가지

교훈 1: "정상 기준"을 알려줘야 한다

❌ 기준 없음

"상태를 분석해주세요" → VLM: "병이 보입니다. 이상 없어요"

✅ 기준 제시

"정상은 매끄럽고 대칭. 이 사진과 비교하세요" → VLM: "울퉁불퉁 → 불량!"

교훈 2: "역할 부여"가 효과적

"당신은 엄격한 품질검사관입니다" → VLM이 더 꼼꼼하게 검사.

교훈 3: "무시할 것"을 명시해야 한다

확대 이미지에서 금형 자국, 엠보싱을 결함으로 오인 → "정상 제조 특성은 무시하세요" 추가.

교훈 4: 여러 위치를 동시에 묻지 않는다

❌ "위 중앙 아래 다 봐"

VLM이 3곳을 동시에 보다가 혼란 → "정상"

✅ "가운데만 봐"

한 곳에 집중 → "찌그러짐 발견!" → 정밀 검사(3등분) 탄생

🔑 9. 핵심 교훈 정리

🔄 순서가 중요하다

VLM에게 좌표를 요구하면 실패. "찾기→분할→판단" 순서로 각 모델이 잘하는 역할만 맡겨야 합니다.

⚡ Open Vocabulary의 힘

YOLOE-26은 80종 고정이 아니라 텍스트로 무엇이든 탐지. "개념 이해"가 "사진 암기"보다 강력합니다.

🎯 SAM3 cutout + 확대

SAM3로 배경 제거 → 물체만 1024px 확대 → VLM이 표면 디테일에 집중. 이 조합이 품질검사의 핵심입니다.

💬 프롬프트 = 검사 기준서

"정상 기준" + "무시할 것" + "찾을 것"을 명확히 지시. 공장 검사 기준서와 같은 역할을 프롬프트가 합니다.

⚖️ 양자화와 해상도의 균형

4bit는 빠르지만 둔감, 8bit는 느리지만 정밀. 1024px가 결함 감지와 오탐 방지의 최적 균형점입니다.

🏭 교실 → 공장

파이프라인 구조는 동일. 카메라(4K) + 모델(72B) + 조명만 업그레이드하면 산업 현장에 바로 적용 가능합니다.
