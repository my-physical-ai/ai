▶ 이 단계가 가장 까다로워요

FLIR ONE은 일반 웹캠이 아니라서, 전용 드라이버(flirone)로 영상을 가상 카메라(/dev/video3)에 흘려보내야 해요.

📌 4-1. 처음 한 번만 — 드라이버 설치
# ① 빌드 도구 + 라이브러리 설치 $ sudo apt update $ sudo apt install -y git cmake build-essential \ libusb-1.0-0-dev libjpeg-dev v4l-utils # ② flirone-v4l2 드라이버 소스 받기 $ cd ~ $ git clone https://github.com/fnoop/flirone-v4l2.git $ cd flirone-v4l2 # ③ 컴파일 $ make # ④ 가상 카메라 모듈 설치 $ sudo apt install -y v4l2loopback-dkms v4l2loopback-utils
📌 4-2. FLIR이 USB로 잡혔는지 찾기
# FLIR ONE Gen2 USB ID는 09cb:1996 $ lsusb -d 09cb:1996 Bus 001 Device 005: ID 09cb:1996 FLIR Systems # 보이면 인식 성공 # 안 보이면 → USB 다시 꽂고 위 명령 재시도
📌 4-3. 가상 카메라 2개 만들기 (video2, video3)
# 기존 모듈 제거 후 새로 2개 생성 $ sudo modprobe -r v4l2loopback $ sudo modprobe v4l2loopback devices=2 video_nr=2,3 \ card_label="FLIR_THERMAL","FLIR_VISIBLE" exclusive_caps=1 # 생성 확인 — video2, video3이 보여야 함 $ ls /dev/video* /dev/video2 /dev/video3
📌 4-4. flirone 실행 (이 터미널은 계속 켜둠)
$sudo ./flirone ./palettes/Grayscale.raw
1"Successfully claimed interface 0,1,2"가 나오면 FLIR 연결 성공이에요.
2이 터미널은 끄지 말고 계속 켜두세요. 끄면 video3도 사라져요.
3팔레트: Grayscale.raw(AI 분석용) / Iron_Black.raw(발열 확인용).
🔧"Assertion `fdwr1 >= 0' failed" 에러가 나면: 가상 카메라가 막힌 거예요. 아래 4-7로 정리하고 4-3부터 다시 하세요.
📌 4-5. 팔레트 바꿔보기 (다양한 색으로 확인)

열화상은 온도 숫자 덩어리예요. 팔레트(색칠 규칙)를 바꾸면 같은 온도라도 완전히 다른 색으로 보여요. 목적에 맞게 골라 쓰세요.

# 사용 가능한 팔레트 목록 + 목적별 추천 보기 $ python test_palette.py --list # 특정 팔레트로 실행 (예: 현장 점검용) $ python test_palette.py --run Iron_Black # 추천 4종을 차례로 비교 (Enter로 넘김) $ python test_palette.py --tour
🔥Iron_Black — 현장 점검용. 발열 지점이 또렷하게 보여요.
🌈Rainbow — 발표·시연용. 작은 온도 차이도 눈에 확 띄어요.
🤖Grayscale — AI 학습용. YOLO·SAM이 분석하기 깔끔해요(app.py 권장).
💡핵심: 파이프라인(YOLO→SAM3→VLM)을 돌릴 땐 Grayscale이 가장 좋아요. AI가 색에 헷갈리지 않고 밝기(온도)만 봐요. 발표할 때만 Iron_Black/Rainbow로 바꾸면 보기 좋고요.
📌 4-6. 영상이 진짜 나오는지 테스트 (권장)
# 방법 ①: 터미널에서 양쪽(video2+video3) 빠른 진단 $ python test_thermal_camera.py --check ✅ /dev/video2: 5/5 수신, 크기 160x128, 🎨 컬러 ✅ /dev/video3: 5/5 수신, 크기 160x128, ⬜ 흑백(회색조) # 방법 ②: 브라우저로 두 영상 나란히 보기 $ python test_thermal_camera.py # → http://NUC_IP:5050 접속하면 video2와 video3이 나란히 표시됨
1--check: video2·video3 둘 다 확인하고, 각각 흑백인지 컬러인지 알려줘요.
2옵션 없이 실행: 포트 5050으로 두 영상을 나란히 보며 어느 게 쓸 영상인지 확인해요.
3영상 위 라벨에 GRAY/COLOR가 표시돼요. app.py가 쓰는 건 보통 흑백(video3)이에요.
💡왜 둘 다 보나: flirone은 video2·video3 두 개를 만들어요. 어느 쪽에 흑백/컬러가 나오는지 눈으로 확인해야 config의 device_index를 맞출 수 있어요.
📌 4-7. 카메라가 막혔을 때 — 프로세스 죽이기
# ① /dev/video3을 누가 쓰는지 확인 (숫자가 PID) $ sudo fuser /dev/video3 /dev/video3: 14440 # ② 그 프로세스가 뭔지 확인 $ ps aux | grep 14440 # ③ 점유 프로세스 강제 종료 (한 줄로) $ sudo kill $(sudo fuser /dev/video3 2>/dev/null | awk '{print $1}') # ④ 이전 app.py가 잡고 있는 경우도 정리 $ sudo pkill -f app.py # ⑤ 그 다음 4-3(modprobe) → 4-4(flirone) 다시 실행
💡왜 막히나: 터미널을 Ctrl+C 없이 그냥 닫거나, 이전 app.py가 카메라를 쥐고 있으면 발생해요. 위 순서로 정리하면 풀립니다.

🎓 핵심 정리

이 실습의 5가지 교훈

1박스보다 텍스트 — 사람처럼 명확한 대상은 "person" 텍스트 프롬프트가 박스보다 정확해요.
2flirone 먼저 — 카메라(video3)가 떠 있어야 app.py가 영상을 받아요. "못 연다" 에러는 flirone부터 확인.
★테스트로 원인 구분 — test_thermal_camera.py로 카메라만 먼저 확인하면, app 문제와 카메라 문제가 바로 갈려요.
🎨팔레트로 골라보기 — test_palette.py로 여러 색을 시험해요. AI는 Grayscale, 발표는 Iron_Black/Rainbow.
3막히면 프로세스 죽이기 — fuser로 찾고 kill로 정리한 뒤 다시 시작.
4config 폴더 필수 — app.py는 config/config.yaml을 찾아요. 없으면 시작도 못 해요.
5서버 먼저, NUC 나중 — SAM3(4090)와 InternVL(0.36) 두 서버를 먼저 켜고, 그다음 NUC 작업.
